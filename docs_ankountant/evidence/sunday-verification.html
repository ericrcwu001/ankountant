Model Descriptions

Memory

Memory estimates recall on study-pile cards. The backend emits point, range, coverage, confidence, reasons, and update time. The model evidence fixture evaluates per-topic trailing recall predictions against held-out recall outcomes and reports calibration, Brier score, and log loss. Open the one-page handout.

Performance

Performance uses sealed exam-style items, including confusion choices and TBS-style graded attempts, rather than study-pile recall. Attempts are logged as ordinary sync-safe notes. Timing, partial credit, and confidence can drag the score down. Performance abstains when evidence is insufficient. Open the one-page handout.

Readiness

Readiness maps sealed performance to a rough CPA 0-99 projection using the documented ADR 0005 monotonic transform. It reports a Wilson-derived range, coverage, confidence, reasons, last updated time, and no score below the give-up rule: fewer than 20 sealed attempts or less than 60% topic coverage. Open the one-page handout.

Rust Change Note

A2 latency-aware too-easy defunding belongs in Rust

The A2 change runs inside the shared scheduler path: a stable rote card answered fast, correct, and confident gets a compact te custom-data flag and a lower desired-retention input before FSRS computes the next interval. This belongs in Rust because desktop and iOS both use the same scheduling engine, undo transactions, card storage limits, FSRS interval math, and sync-safe card state. A Python-only or Swift-only implementation would drift between clients and could not reliably preserve Anki undo semantics.

Main upstream-touching files: rslib/src/ankountant/defund.rs, rslib/src/ankountant/logic.rs, rslib/src/ankountant/schedule.rs, rslib/src/ankountant/tests.rs, scheduler answer/preview call sites, Python bridge tests, and iOS FFI/review tests. Merge difficulty is moderate: the change is mostly additive under rslib/src/ankountant/, but future upstream scheduler changes around desired retention, answer_card, card custom data, or undo boundaries must be rechecked.
